How would you design a python 3.11 framework that streams data from one data source to a data destination?
Either data source or destination can be a Database or an API.
I am attaching an example JSON that would be used to define available data streams in either source or destination. The endpoint catalog must define which endpoints are providing data, which ones are consuming data and which ones can do both.
The catalog should also define which support full refresh and which support incremental refresh.
There would also be a JSON mapping file that maps the source to a destination.
Once that mapping is in place, what would be the best mechanism and process to quickly and reliably move data from source to destination?
I would like the process to be fault-tolerant, fast and reliable.


List of problems that this framework should address:
| Problem                        | Mitigation                                                                                                                                                                     |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Process crash / network outage | Offsets stored in `state.json`. On restart the Engine resumes from last committed LSN/timestamp.                                           |
| Destination unavailable        | `async_retrying.Retrying` wrapper with *decorrelated jitter* back-off; circuit breaker trips after N minutes.                                                                  |
| Schema drift                   | Each `discover()` call generates a schema hash; if the hash changes, the Engine calls `prepare()` again, which can `ALTER TABLE … ADD COLUMN` or create a new versioned table. |
| Poison record                  | Row-level `try/except`, bad rows sent to a DLQ (e.g. Kafka topic or a simple `deadletter/` folder).                                                                            |
| Duplicate writes               | Destination connectors must implement **idempotency**. For DBs: `INSERT … ON CONFLICT DO UPDATE`. For REST APIs: `Idempotency-Key` header.                                     |
